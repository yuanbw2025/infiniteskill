# InfiniteSkill：把一本书变成 AI 的超能力 —— 我做了一个「文档→技能」编译器

## 一、背景：AI 读书的困境

大模型时代，我们已经习惯了把文档丢给 AI 让它帮忙总结、提炼。但你有没有遇到这样的问题——

你把一本 300 页的《项目管理知识体系指南》丢给 ChatGPT，让它帮你做项目规划。它给出了一个看起来不错的回答，但：

- **信息丢失严重**：书里那些精妙的流程、矩阵、检查清单，AI 只记住了皮毛
- **无法复用**：下次再问，AI 又从零开始"读"
- **缺乏结构**：AI 把所有知识揉成一团，分不清哪些是决策技能、哪些是执行流程

**根本原因是：大模型的 Context Window 是一次性的，而专业书籍的知识是结构化的。**

这就像你让一个实习生用 5 分钟翻完一本教材然后立刻上岗——他能记住多少？

## 二、核心思路：编译，而非对话

我换了一个思路：**不是让 AI 在对话中临时"阅读"，而是提前把书"编译"成 AI 能直接调用的技能模块。**

这个思路来源于软件工程中的一个基本概念——**编译器**。

```
传统编译器：  源代码(.c)  → 编译 → 可执行文件(.exe)
InfiniteSkill：专业书籍(.pdf) → 编译 → 技能包(.zip)
```

**编译和对话的本质区别：**

```
┌─────────────────────────────────────────────┐
│         对话模式（传统 RAG）                   │
│                                              │
│  📚 整本书 → [Context Window] → 🤖 回答      │
│                                              │
│  问题：上下文有限、信息压缩、不可复用           │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│         编译模式（InfiniteSkill）              │
│                                              │
│  📚 整本书 → [编译引擎] → 📦 结构化技能包     │
│                           ↓                  │
│               🤖 智能体直接调用（精准·可复用）  │
│                                              │
│  优势：离线编译、结构化存储、精准调用           │
└─────────────────────────────────────────────┘
```

## 三、编译流水线：6 步把书变成技能

InfiniteSkill 的编译流水线分为 6 个阶段，我画了一张系统架构图：

```
┌──────────────────────────────────────────────────────────┐
│                 InfiniteSkill 编译流水线                    │
├──────────┬───────────┬───────────┬───────────┬───────────┤
│  Stage 1 │  Stage 2  │  Stage 3  │  Stage 4  │  Stage 5  │
│  文档解析 │  语义拆解  │  技能提取  │  逻辑建模  │  团队设计  │
│          │           │           │           │           │
│ PDF/EPUB │ 4000字/块  │ AI提取    │ 合并去重   │ 多角色    │
│ TXT/MD   │ 语义边界   │ 原子技能   │ 建立依赖   │ 分配技能   │
│     ↓    │     ↓     │     ↓     │     ↓     │     ↓     │
│  纯文本   │  N个语块  │  M个技能   │  K个核心   │  Agent    │
│          │           │           │  技能      │  Team     │
├──────────┴───────────┴───────────┴───────────┴───────────┤
│                        Stage 6                            │
│                      技能封装(ZIP)                         │
│  SKILL.md + agents.json + skills/*.md → 📦 Skill Pack    │
└──────────────────────────────────────────────────────────┘
```

**每个阶段做了什么：**

**① 文档解析** — 支持 PDF（pdf.js）、EPUB（epub.js）、TXT、Markdown 四种格式，统一提取为纯文本。

**② 语义拆解** — 按 4000 字符的逻辑边界分块。为什么是 4000？因为这个大小既能包含足够的上下文，又不会超出单次 API 调用的效率区间。

**③ 技能提取** — 对每个语义块，调用 Gemini 2.5 Flash 提取"可执行技能"。每个技能包含：ID、名称、触发条件、输入参数、执行逻辑、输出格式、依赖关系。

**④ 逻辑建模** — 调用 Gemini 2.5 Pro（更强的推理能力），将所有原子技能进行：合并冗余、建立依赖、优化命名。

**⑤ 团队设计** — 同样由 Pro 模型完成，把技能分配给不同的"智能体角色"，就像一家公司里不同的专业岗位。

**⑥ 技能封装** — 生成标准的 ZIP 技能包，包含主技能文件、智能体配置、各独立技能的 Markdown 文件。

## 五、技术选型与思考

### 通用多模型支持：用你手头任何一个大模型

InfiniteSkill 的一个核心设计原则是**不绑定任何单一模型提供商**。目前内置支持 9 大 AI 提供商，一键切换：

> **GPT** · **Claude** · **Gemini** · **DeepSeek** · **千问** · **Kimi** · **GLM** · **豆包** · **自定义（任何 OpenAI 兼容 API）**

每家提供商都支持自定义填写具体模型名称，随时切换最新模型，不受版本绑定。

**中国境内的用户**可以直接使用千问、Kimi、智谱 GLM、豆包等国产模型，无需科学上网。海外用户可以用 GPT、Claude、Gemini。

**自定义选项**支持任何兼容 OpenAI `/v1/chat/completions` 格式的服务——包括 vLLM 私有部署、Ollama 本地模型、one-api 中转站等。只要你的服务能响应 OpenAI 格式的请求，InfiniteSkill 就能用。

技术实现上，除了 Claude（走原生 Anthropic Messages API）之外，所有提供商统一通过 `openai` npm 包的 `baseURL` 参数进行适配，一套代码搞定全部。

### 为什么不用 RAG？

RAG（检索增强生成）是目前主流的"让 AI 读书"方案，但它有几个根本问题：

```
RAG 的问题                          InfiniteSkill 的解决方案
─────────────────────                ─────────────────────
检索粒度粗（通常是段落级）    →     技能粒度精确（每个Skill有明确的触发条件）
缺乏逻辑依赖关系           →     自动建立技能间的依赖图
每次对话都要检索            →     编译一次，永久使用
无法多角色协作              →     自动设计多智能体团队
```

**RAG 是搜索引擎思维，InfiniteSkill 是编译器思维。**

### Skill 文件格式

每个技能文件是一个带 YAML 前置元数据的 Markdown 文件：

```yaml
---
id: earned_value_analysis
name: 挣值分析
trigger: 当需要评估项目绩效偏差时
inputs: ["实际成本(AC)", "计划价值(PV)", "挣值(EV)"]
dependencies: ["cost_budgeting", "schedule_estimation"]
---

# 挣值分析

## 触发场景
当需要评估项目绩效偏差时

## 执行逻辑
1. 计算成本偏差 CV = EV - AC
2. 计算进度偏差 SV = EV - PV
3. 计算成本绩效指数 CPI = EV / AC
4. 计算进度绩效指数 SPI = EV / PV
5. 若 CPI < 1.0，发出成本超支警告
...
```

## 六、开源与使用

InfiniteSkill 已完全开源，你可以：

- **在线体验**：直接上传文档，免费编译（每日 80 次额度，走 Gemini 代理通道）

## 七、写在最后

InfiniteSkill 的核心洞察其实很简单：

> **知识的最佳传递形式，取决于接收者。**
> 
> 人类的最佳学习载体是书籍——有上下文、有案例、有叙事。
> AI 的最佳执行载体是结构化技能——有触发条件、有输入输出、有依赖关系。
>
> **InfiniteSkill 就是这两种形式之间的翻译器。**
